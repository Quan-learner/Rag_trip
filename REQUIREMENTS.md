# 项目开发需求文档 - 旅游计划顾问 RAG 系统

## 项目概述
开发一个智能**旅游计划顾问** RAG 系统，前端使用现代化 AI 聊天界面，后端使用 Python 实现完整的 RAG 流程。用户可以与 AI 对话制定旅游计划，并支持知识库管理。

---

## 一、前端要求（Next.js + assistant-ui）

### 技术栈
- Next.js 15 + TypeScript + Tailwind CSS
- 使用最新版 **assistant-ui**（@assistant-ui/react）
- 选择 **cloud 模板**（支持对话持久化）

### 界面要求
- 界面风格：现代、简洁、**暗黑模式优先**（类似 ChatGPT）
- 必须支持：Markdown 渲染、代码高亮、Streaming 输出、复制按钮
- 支持主题切换（明暗模式）
- 预留清晰的后端接口位置，方便后续接入 GLM-5、Claude 或 OpenAI
- 项目结构清晰，包含完整的 README.md 使用说明

---

## 二、后端 RAG 系统要求（Python）

### 技术栈
- LLM：本地部署 **DeepSeek-R1:1.5B**
- Embedding 模型：本地部署 **nomic-embed-text:latest**
- 向量数据库：**ChromaDB**
- Web 框架：**FastAPI**（推荐）

### 阶段 1：准备阶段
- **场景定位**：旅游计划顾问，支持用户自定义添加/管理知识库（景点信息、攻略、预算、注意事项等）
- **文档预处理与 Chunk 策略**：
  - 采用**语义分块 + 固定大小 + 重叠**组合策略
  - 重叠大小：50-100 tokens
  - 块大小建议：512~1024 tokens

### 阶段 2：核心构建阶段
- **检索策略**：
  - 第一层：混合检索（向量检索 + BM25/关键词检索）
  - 第二层：**RRF 融合**（Reciprocal Rank Fusion）
  - 第三层：**Cross-Encoder 重排序**（Top 10 → Top 3）
- 实现知识库管理功能（增删改查文档）

### 阶段 3：生成与优化阶段
- **Prompt 工程**：
  - 构建高质量 System Prompt（角色：专业旅游规划师）
  - 要求 AI 必须引用来源、输出结构化旅游计划
  - 支持上下文压缩（Long Context 优化）
  - 支持引用溯源（返回来源 Chunk）
- **系统集成**：
  - 使用 FastAPI 封装 RESTful 接口（`/chat`、` /upload`、` /documents` 等）
  - 实现文档增量更新机制
  - 添加基本权限控制与安全过滤（防止注入、敏感信息等）
